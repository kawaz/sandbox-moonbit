# DR-011: プロジェクト名の検討

## 経緯

CLI引数パーサを sandbox-moonbit から独立リポジトリに分離するにあたり、プロジェクト名を検討した。

初期の候補出しでは、直近の設計議論で「先食い最長一致」がホットワードだった影響で、「食べる」「貪る」系のメタファー（devour, glutton, chomp, gobble, munch 等）に候補が偏った。これを受けて「特定の設計思想に縛られない」を命名条件に追加し、道具/素材・神話・他言語借用・造語など幅広い方向から再検討した。本来はもう少し多角的に候補を深掘りしたかったが、kuu が条件を高水準で満たしていたため早期に決定した。

## 命名条件

- **moon prefix 不可**: MoonBit 公式ツールとの誤認リスク、多言語展開に不向き
- **設計思想に縛られない**: 「先食い」「greedy」等は設計の一過性ホットワードの可能性
- **短い・覚えやすい・ユニーク**
- **多言語展開しても違和感なし**: コアは MoonBit だが他言語にも展開予定
- **conflict が低い**: GitHub、npm、crates.io、PyPI、mooncakes.io 等で競合しない

## 決定

**`kuu`** を採用。

- フィンランド語で「月」(= MoonBit 出自のさりげない暗示)
- 日本語で「食う」(= 引数を消費するパーサの動作)
- 3文字で最短、全レジストリ空、conflict 極低

## 検討した候補

### conflict 低（有力候補）

| 名前 | 由来 | conflict | コメント |
|---|---|---|---|
| **kuu** | フィンランド語「月」+ 日本語「食う」 | 極低 | 採用。3文字、二重の意味、全レジストリ空 |
| jaki | フィンランド語「分ける/割る」 | 極低 | パーサの動作に直結、kuuとフィンランド語つながり |
| pilkka | フィンランド語「コンマ/区切り」 | 極低 | 意味は完璧（区切り = CLI引数の本質）。6文字でやや長い |
| valka | 古ノルド語「選ぶ」(Valkyrie語根) | 極低 | 「選択」がオプション解析に合う |
| ordo | ラテン語「秩序/順序」 | 低 | 堂々とした響き。npm に非活発パッケージあり |
| norn | 北欧神話の運命の女神 | 低 | 神秘的。npm に非活発パッケージあり |
| margs | moon + args | 低 | 短くて覚えやすい。moon を強く押す必要がないため優先度下がる |
| optcraft | option + craft | 低 | 完全クリーンだが意図が伝わりにくい |
| moonargs | moon + args | 低 | 完全クリーンだが moon prefix 問題 |
| moonbite | MoonBit + bite | 低 | 完全クリーンだが公式誤認リスク |

### 惜しいが注意が必要

| 名前 | 由来 | conflict | コメント |
|---|---|---|---|
| glutton | 大食い = 先食い | 低〜中 | Go製ハニーポットと被る。メタファーは秀逸だが設計思想に縛られる |
| greed | greedy(先食い) | 中 | 設計思想を直接表現。crates.io にトレードツール。設計思想に縛られる |
| devour | 貪り食う | 中 | npm に別用途で存在 |
| claw | 掴み取る、clap の対抗馬的響き | 中 | crates.io で claw-* が活発 |
| toku | 日本語「解く」 | 中 | tokio との混同リスク |
| crescent | 三日月 | 中 | 一般名詞で散在 |
| bite | 噛む | 中 | PyPI に bite-parser（パーサ系）あり |
| cleave | 割る/分割する | 中 | cleave.js が有名 |
| sana | フィンランド語「言葉」 | 中 | crates.io にレキサ/パーサ系あり |

| lunarg | Vulkan SDK 開発企業名。商標リスク |

| munch | 3レジストリ全てで衝突。PyPI の munch は広く使われている |
